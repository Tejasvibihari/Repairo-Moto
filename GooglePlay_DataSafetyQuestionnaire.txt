3. PERMISSIONS USAGE
==============================================

3.1 CAMERA Permission (android.permission.CAMERA)
- Purpose: Capture vehicle before/after images (mechanics), profile image upload (customers)
- When Used: Only when user initiates image capture or selection
- Data Retention: Images retained for 7 days (service docs) or indefinite (profile)
- Justification: Essential for service documentation and dispute prevention
- Optional: Camera permission is optional; revoking may affect mechanic's ability to document service

3.2 LOCATION Permission (android.permission.ACCESS_FINE_LOCATION - Foreground only)
- Purpose: Determine customer location for service booking and mechanic dispatch
- When Used: ONLY when user is actively using the app to book or track service
- NO Background Location: Zero background tracking, passive surveillance, or continuous monitoring
- Data Retention: Location is NOT permanently stored; only used for active dispatch
- Accuracy: Approximate location (±500m radius) is sufficient; precise GPS not required
- Justification: Essential for on-demand service delivery and mechanic routing
- Optional: Location can be disabled; user can enter address manually (less convenient)

3.3 POST_NOTIFICATIONS Permission (android.permission.POST_NOTIFICATIONS)
- Purpose: Send service notifications (booking confirmation, mechanic arrival, payment reminders, promotions)
- When Used: Throughout the user session
- Channels: Service updates (critical), Promotional (optional)
- Data Retained: Notification tokens retained while app is installed
- Justification: Essential for real-time service communication and customer engagement

3.4 INTERNET Permission (android.permission.INTERNET)
- Purpose: Connect to Repairo Moto servers for bookings, payments, and support
- When Used: Whenever app needs to sync data or communicate with backend
